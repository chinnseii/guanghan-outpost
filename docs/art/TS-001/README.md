## Intent

TS-001 定义《广寒前哨》第一张目标画面：玩家第一次踏上月球，运输船在身后准备离开，旧基地在远方发出微弱暖光，地球位于画面视觉中心。

这不是要在 Godot 里写死的剧情脚本，而是后续美术、镜头、UI 和第一小时体验的共同目标。

## Composition

- 地球是第一视觉中心，必须清晰、饱和、偏蓝。
- 玩家位于画面左下区域，体量很小，突出孤独。
- 运输船在玩家身后，表示地球支援即将离开。
- 旧基地在远方，暖光微弱，表示希望和方向。
- 远处基地与近处补给/设备之间有路径感，让玩家知道“我要走过去”。

## Color Script

- 月球表面：冷灰、黑、低饱和。
- 基地灯光：暖黄、琥珀色，代表人类生活。
- 工程设备：低饱和灰、白、少量橙色标识。
- 地球：高饱和蓝，是全画面最有生命感的颜色。
- 绿色：不出现。第一小时早段不应该让月面上出现绿色。

## Lighting

- 月面整体偏冷。
- 基地、运输船和设备灯使用暖光。
- 地球可以作为远处的情绪高光。
- 玩家附近的光源应该小而克制，不要把月面照得像地球夜市。

## UI Direction

- HUD 极简，优先显示宇航服状态、时间和环境信息。
- UI 风格克制、工程化、低透明深色面板。
- UI 不应该抢过地球和月面场景的视觉中心。
- 第一幕里避免大型任务面板持续占屏，允许沉默和凝视。

## Implementation Notes

不要为了 TS-001 硬编码一次性逻辑。应该通过可复用系统实现：

- Game State Machine：管理“第一次踏上月球”“运输船离开”“进入旧基地”等体验状态。
- Fixed Camera / Cinematic Camera：支持固定镜头和轻量镜头切换。
- Lighting System：支持冷月面、暖基地灯、设备灯和室内灯。
- UI Manager：支持极简 EVA HUD 和剧情时隐藏普通经营 UI。
- TileMap Layer Framework：支持月面、旧基地、设备、前景装饰分层。

## Do Not Drift

- 不要把这一幕做成采矿教程。
- 不要在第一幕放出复杂建造、科技树和机器人自动化。
- 不要把绿色到处铺开。
- 不要用弹窗和箭头破坏“第一次被留下”的沉默。
