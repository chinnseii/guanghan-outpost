# 月面地图 · 设计/架构文档

状态：**设计已定稿（决策已拍板），尚未实现**。作者：Claude Code（代 Codex），2026-07-08。
定位：一张长期大地图 + 活动半径由装备/基地能力决定。第 6 大项决策已与人类确认（见第 11 节"决策记录"）。开工从"阶段1 种子"开始。

---

## 1. 目标与范围

- **一张长期存在的大地图**，玩家实际活动半径由**装备（宇航服 / 月球车）+ 基地能力（中继 / 前哨 / 副基地）**决定。
- 进度分层：
  - **前期**：初代宇航服 + 步行 → 近基地区（**且不可夜间出行**）。
  - **中期**：月球车 → 远端、矿点。
  - **后期**：升级宇航服 / 月球车 + 中继 / 前哨 / 副基地 → 远区、旧前哨、副基地建设（**升级服后可夜间出行**）。
- 两条铁律：
  1. **不做一次性小地图**。从近基地区域开始增量搭建最终大地图的**第一个区域**，架构上能往外扩，扩展 = 加区域，不返工。
  2. 活动半径 = **资源预算 + 安全返回锚点**，不是硬墙（第 5 节）。

## 2. 两种地图模式

| 模式 | 现状 | 本方案 |
|---|---|---|
| **基地/舱室内部** | 分房间 Control blockout（`training_base_map`/`sprint06_base_scene`） | **保持房间制不变** |
| **月面地表** | 无 | **新增：TileMap 世界 + 迷雾 + 可达半径** |

**边界 = 气闸**。`DoorStateManager` 已预置 `airlock_room → lunar_surface_near_base`（外门 `requires_suit_to_pass=true`）。气闸是室内↔地表的模式切换点。

## 3. 世界坐标、区域与 POI

- **单一世界坐标空间**（TileMap，见第 7 节），基地在原点附近。所有地表内容挂在这个坐标系里。
- **区域（Region）**是内容组织单位，不是硬边界：
  ```
  Region { region_id, display_name, center:Vector2, radius:float,
           band: near/mid/far, points_of_interest:[poi_id], discovered:bool }
  ```
- **POI 分层**（内容投放节奏）：
  - **前期（near）**：**月球冰采集点**、**月壤采集点**（采集→回收进背包→带回基地）。
  - **后期（mid/far）**：**矿坑**、**永久日照峰**、**永久黑夜陨石坑**、**废弃基地**。
  - 注：永久日照峰 / 永久阴影坑是**真实月球地貌**（月球两极附近，如 Shackleton 坑缘有近乎持续日照的峰、坑底是永久阴影冷阱、含水冰）——正好接电力（日照峰=理想太阳能/中继址）与水资源（阴影坑=水冰）玩法，科学上站得住。
- **地图尺寸（按氧气往返预算定，不按像素）**：
  - 真正约束是"初代服氧电够不够安全往返"。建议**近区边缘 POI ≈ 初代服安全单程可达**。
  - 起步参考值（先跑再调）：近区约 **64×64 tile**（64px/tile ≈ 4096×4096px，约 5–6 个屏宽）；整图按 near→mid→far 三带，大致 **10–20× 近区**每轴。**这些数字靠阶段1 试玩用氧电预算标定**，不是最终值。
- **可扩展性（关键，别纠结总大小）**：地图**不会一做就固化**。Godot TileMap 的格子是**稀疏、无上界**的坐标网格，任何时候都能在任意坐标继续加 tile 区块 + 区域数据 + POI——加地表是**追加，不是重建**；迷雾按区域存，加区域只多几条记录；可达半径是资源制，跟地图边界无关。
  - **策略**：近区**小而密**起步（不空），之后**一带一带往外长**，有内容才加区域 → 不会太空、也放得下后期想法。
  - **唯一建议早期定死的是"坐标尺度"**（多少世界单位 = 1 分钟氧气步行距离），因为内容都按它摆；而地图**能走多远（extent）保持开放**。阶段1 只标定尺度、近区做小即可。
  - 若后期整图变得很大导致性能问题，再上**分块/流式加载**优化即可，仍不需要重建。

## 4. 迷雾 / 探索状态

- 每个区域/POI 有 `discovered`，随实际到达点亮，**持久化**。
- **初期地图全黑**，到达过的地方才显示（见第 9 节地图 UI）。
- 迷雾只管"信息可见性"，不做硬墙——能不能去由第 5 节资源预算决定。

## 5. 可达半径：资源预算 + 安全返回锚点（核心）

可达半径 = "带着现有氧电，能走多远还能安全回到最近补给锚点"。复用 `SuitManager`。

### 5.1 复用的 SuitManager 现有能力
- `suit_oxygen`/`suit_oxygen_capacity`、`suit_power`/`suit_power_capacity`：EVA 资源池。
- `can_start_eva()`：氧电低于 `MIN_EVA_OXYGEN`/`MIN_EVA_POWER` 不许出舱。
- `consume_suit_resources(minutes, activity_type)`：随时间/移动消耗氧电。
- `suit_speed_multiplier`/`get_actual_minutes()`/`upgrade_suit_speed()`：服等级→速度→单位距离耗时→可达远近。
- `refill_suit_oxygen()`/`recharge_suit_power()`/`service_suit_full()`：锚点补给。
- `suit_level`：服等级，用于**夜间出行门槛**（见 5.4）。

### 5.2 预算与安全返回
- **安全返回锚点**：气闸、（后期）中继站/前哨/副基地、月球车（移动锚点）。到达→补给氧电→重置预算。
- **可达半径**（UI 实时算）：当前氧电 − "从当前点返回最近锚点所需氧电" = 可继续外出的余量 → 换算成还能走多远。
- **安全返回点警告**：走到"再走就回不来"的临界，HUD 明确提示"该返航"。
- **月球车**：更大氧电储备 + 更快 + 移动锚点 → 预算大幅提升，够到 mid/far。可进化（见 5.5）。
- **中继/前哨/副基地**：新固定锚点 → 半径一段段外推。

### 5.3 超预算 / 遇险后果（已定）
**不做瞬间死亡**，而是重罚 + 救援：
- **背包物资全部掉落**在倒下地点，形成一个**可回收掉落点**（回去捡得回来）。
- **玩家在基地醒来**，四项健康数值（精力/饱腹/营养/心理）**强制设为 30**（稳住但重度透支）。
- **合理理由（已定）**：宇航服带**应急信标**，氧气/生命体征触及临界自动触发；基地派出**救援无人车**循信标找到昏迷者，送回气闸/医疗位。应急回收**优先救人**，未固定的背包物资**被留在原地**（∴可回收）。
- **代价含基地电量（已定）**：派出救援无人车消耗一笔**基地电量**（`PowerSystemManager` / `BaseStatusManager.power`）——遇险不只是个人损失，也拖累基地补给，强化"别乱冒险"的压力。
- 逐级 HUD 警告在触发前给足提示（低氧/低电/临界返航）。
- 施加方式：走 `PenaltyManager` 统一分派——掉物 remove_items→掉落点、健康四项设 30、**基地电量扣减**、可扣时间。

### 5.4 月夜耦合（已定）
`TimeManager` 已有月昼/月夜/月夜末期。
- **前期（初代服）**：**月夜不可出行**——月夜期 `can_start_eva()` 相关门槛拒绝出舱（气闸外门锁）。
- **后期（升级服，`suit_level` 达标）**：解锁夜间出行；夜间氧电/保温消耗更高、风险更大。
- 月夜=无太阳能、更冷，天然强化"夜间更危险"的体验。

### 5.5 月球车进化（已定）
纯载具 → 逐级升级 → "车上小基地"：
- L1 纯载具：提速 + 移动锚点（可补一点氧电）。
- L2+：加储物、加补给/维修、更大氧电缓冲，逐步变成移动的安全据点。

## 7. 技术选型（已定：TileMap）

- **地表用 TileMap**（地形/纹理），世界坐标节点放 POI/锚点/玩家，迷雾用覆盖层。
- **后期地貌**：高山、月球峡谷——用 TileMap 的**地形层 + 通行性**表达（山/峡谷可阻挡或引导路线、影响中继视线/选址）。这要求坐标系与通行性从一开始就为高程/障碍留好扩展位。
- 室内继续房间制；气闸切换模式。扩展 = 往同一坐标系加 tile 区块 + POI，不返工。

## 8. 与现有系统对接

| 系统 | 用途 |
|---|---|
| `SuitManager` | 氧电预算、出舱门槛、速度、锚点补给、夜间门槛（`suit_level`） |
| `TimeManager`/`MovementTimeManager` | 地表移动推进时间；月昼/月夜门槛与风险 |
| `DoorStateManager` | 气闸=室内↔地表边界（已预置 `lunar_surface_near_base`）；夜间锁外门 |
| `TaskManager`（新） | 探索/采集/前哨作为任务进目录 |
| `PenaltyManager`（新） | 遇险后果统一分派（掉物/健康设 30/扣时间） |
| `SupplyManager` | 后期副基地/前哨物资，可挂补给排期 |
| `PlayerStateManager` | 当前区域/是否地表/是否穿服（已有 exterior 概念） |
| `HealthManager` | 遇险醒来后的健康数值（设 30）、危险态 |
| `BackpackManager` | 采集物入背包、遇险掉物、回收 |

## 11. 决策记录（人类已拍板 2026-07-08）

1. **超预算后果**：重罚——背包物资全掉落（可捡回）+ 玩家在基地醒来 + 四项健康强制 30。回基地理由=**应急信标 + 基地派救援无人车回收**；派车**额外消耗基地电量**（→ 5.3）。
2. **技术选型**：TileMap，后期加入高山与月球峡谷。
3. **月夜**：前期不可夜间出行，后期升级服解锁。
4. **POI**：前期月球冰/月壤采集；后期矿坑、永久日照峰、永久黑夜陨石坑、废弃基地。
5. **月球车**：可进化，纯载具→"车上小基地"。
6. **地图 UI**：做；初期全黑、到达点亮、玩家可主动标记。
7. **地图尺寸/扩展**：**不预先定死总大小**。近区小而密起步，后期一带带往外扩（TileMap 稀疏无上界，追加不重建）；**只早期定死坐标尺度**，extent 开放（→ 第 3 节可扩展性）。

**仍需你/GPT 终确认的小项**：
- 第 3 节坐标尺度与初代服氧电往返预算的具体数值（阶段1 试玩时标定）。
- 第 9 节：全屏地图之外，是否也要常驻小地图（minimap）。

## 12. 增量落地路线

- **阶段1（种子，可玩）**：近基地地表**一个 TileMap 区域** + 气闸出入 + 氧电**可达半径 UI/警告** + 回气闸补给 + 遇险救援（掉物/醒来/健康30）雏形。标定氧电预算与近区尺寸。
- **阶段2**：迷雾/区域数据 + 地图面板（全黑→点亮→标注）+ 第 1 个采集 POI（月球冰/月壤→背包→带回）。
- **阶段3**：月球车 L1（提速/移动锚点）→ 打开 mid 区。
- **阶段4**：月夜门槛 + 服/车升级 + 中继/前哨/副基地锚点 → far 区、日照峰、阴影坑、废弃基地。
- 每阶段单独验证 + 提交。
