# Prologue Text Style Guide

Status: Sprint 08.6 baseline

## Voice

Use a quiet mission-system voice:

- precise
- restrained
- procedural
- engineering-oriented
- emotionally controlled

The system should feel like a reliable mission computer, not a mascot, narrator, or motivational coach.

## Approved Terms

- 状态
- 归档
- 任务建议
- 最低稳定
- 生命支持
- 驻留记录
- 系统检查
- 复核
- 恢复
- 维持阈值
- 通信延迟
- 地面任务组

## Avoid

- 冒险
- 勇者
- 拯救世界
- 恭喜
- 太棒了
- 可爱
- 魔法
- 奇迹
- 超级
- 奖励
- 关卡

## Objective Wording

Prefer short verb-object objectives:

- 提交申请
- 查看资格初审结果
- 进入国家训练序列
- 完成宇航服基础控制训练
- 查看任务派遣通知
- 进入旧基地
- 恢复基础供电
- 恢复最低生命支持
- 查看旧温室
- 稳定最后一株植物
- 发送对地报告
- 返回居住舱休息

## Report Style

Reports should read like mission logs, not quest completion text.

Example:

```text
广寒前哨 D06 驻留报告

温室生命信号：稳定
植物恢复迹象：微弱
补光输出：低功率稳定
水循环：最低维持

备注：最后一株植物出现轻微恢复迹象。
建议继续维持当前环境。
```

## Last Plant Rule

The Last Plant should remain central, but the text should not anthropomorphize it.

Use:

```text
植物生命信号：Stable。
恢复幅度：有限。
继续观察。
```

Avoid:

```text
小植物终于开心起来了！
```

## Narrative Black Screens

Black screens should show only centered narrative/system text.

Hide gameplay HUD, checklists, minimap, resources, and dev labels.
